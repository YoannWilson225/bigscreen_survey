Bigscreen API Documentation

Listes des questions: /api/questions

Description: Cette fonction se charge de récupérer toutes les questions.

Méthode: GET

Paramètres:

  • 'questions' (array): Tableau retournant les questions.
  • 'message' (string): Message.

Response: JSON object avec les propriétés suivantes:

  • id (integer): ID de la question.
  • body (string): Corps de la question.
  • type (string): Type de la question.
  • options (array): Tableau contenant les options de la question (uniquement pour les questions de type 'A').

Stockage des réponses: /api/answers

Description: Fonction de stockage des réponses.

Méthode: POST

Paramètres:

  • 'answers' (array): Tableau contenant les réponses aux questions du sondage.

Response: JSON object avec les propriétés suivantes:

  • 'visitor_id' (string): Clé de référence unique associée aux réponses enregistrées.

Réponses du sondage par visitor_id: /api/showAnswers/{visitorId}

Description: Récupère toutes les réponses du sondage associées au visitor_id spécifié.

Méthode: GET

Paramètres:

  • 'visitorId' (string): Clé de référence unique du visitor.

Response: JSON object avec les propriétés suivantes:

  • 'status' (integer): Statut de la réponse (200 pour succès, 404 pour échec).
  • 'responses' (array): Tableau contenant les réponses associées au visitor_id spécifié.

Toutes les réponses du sondage: /api/admin/answers/get/{token?}

Description: Récupère toutes les réponses du sondage.

Méthode: GET

Paramètres:

  • 'token' (string, optionnel): Jeton d'authentification de l'administrateur.

Response: JSON object avec les propriétés suivantes :

Connexion admin: /api/admin/login

Description: Connecte un nouvel utilisateur administrateur.

Méthode: POST

Paramètres:

  • 'email' (string): Adresse e-mail de l'administrateur.
  • 'password' (string): Mot de passe de l'administrateur.

Response: JSON object avec les propriétés suivantes :

  • 'error' (string, optionnel): Message d'erreur en cas d'échec de la connexion.
  • 'token' (string): Jeton d'authentification de l'administrateur en cas de succès.
  • 'adminId' (integer): ID de l'administrateur en cas de succès.
  • 'message' (string): Message de succès en cas de succès.
  • 'status' (string): Statut de la réponse ('failed' en cas d'échec, 'done' en cas de succès).
